# 文档模板使用指南

本目录包含用于创建项目文档的标准模板。使用这些模板可以确保文档的一致性和完整性。

## 模板列表

### 1. template-getting-started.md
**用途**：入门文档模板  
**适用于**：
- 安装指南
- 快速开始教程
- 新手入门文档

**特点**：
- 强调前置条件检查
- 提供清晰的步骤说明
- 包含常见问题解答
- 引导用户进行下一步

### 2. template-user-guide.md
**用途**：使用文档模板  
**适用于**：
- 功能说明文档
- API 参考文档
- 配置指南
- 示例集合

**特点**：
- 结构化的功能说明
- 详细的参数和返回值说明
- 丰富的代码示例
- 配置选项表格

### 3. template-development.md
**用途**：开发文档模板  
**适用于**：
- 开发环境搭建指南
- 贡献指南
- 代码规范文档
- 架构设计文档

**特点**：
- 详细的环境搭建步骤
- 项目结构说明
- 开发流程指导
- 代码规范示例

### 4. template-advanced.md
**用途**：进阶文档模板  
**适用于**：
- 设计原理解析
- 性能优化指南
- 故障排查手册
- 深度技术讲解

**特点**：
- 核心概念深入讲解
- 最佳实践对比
- 性能优化案例
- 故障排查流程

## 使用方法

### 步骤 1：选择合适的模板

根据要创建的文档类型，选择对应的模板：
- 入门类 → `template-getting-started.md`
- 使用类 → `template-user-guide.md`
- 开发类 → `template-development.md`
- 进阶类 → `template-advanced.md`

### 步骤 2：复制模板

```bash
# 示例：创建安装指南
cp docs/zh/templates/template-getting-started.md docs/zh/getting-started/installation.md
```

### 步骤 3：填充内容

1. 替换所有 `[占位符]` 为实际内容
2. 删除不需要的章节
3. 根据需要添加新章节
4. 确保所有链接正确

### 步骤 4：检查清单

创建文档后，请检查：
- [ ] 标题清晰准确
- [ ] 目录完整（如适用）
- [ ] 所有占位符已替换
- [ ] 代码示例可执行
- [ ] 链接有效
- [ ] 术语使用一致（参考 glossary.md）
- [ ] 中英文之间有空格
- [ ] 标点符号使用正确
- [ ] 更新日期和版本信息

## 模板定制

### 添加新章节

如果需要添加模板中没有的章节，请遵循以下格式：

```markdown
## 章节标题

[章节内容]

### 子章节标题

[子章节内容]
```

### 代码块格式

使用语法高亮的代码块：

````markdown
```python
# Python 代码示例
def example():
    pass
```

```bash
# Bash 命令示例
echo "Hello"
```
````

### 表格格式

使用 Markdown 表格：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

### 提示框

使用 emoji 和格式化文本创建提示框：

```markdown
💡 **提示**：这是一个有用的提示

⚠️ **注意**：这是需要注意的事项

❌ **错误**：这是常见错误

✅ **推荐**：这是推荐做法
```

## 文档编写规范

### 1. 语言规范

- 使用简体中文
- 保持专业、清晰、简洁
- 避免口语化表达
- 使用主动语态

### 2. 格式规范

- 标题使用 `#` 标记，最多 6 级
- 代码使用反引号 `` ` `` 或代码块
- 列表使用 `-` 或 `1.` 标记
- 强调使用 `**粗体**` 或 `*斜体*`

### 3. 术语规范

- 参考 `docs/zh/glossary.md` 使用统一术语
- 专有名词首次出现时提供英文原文
- 保持术语使用一致性

### 4. 链接规范

- 内部链接使用相对路径
- 外部链接使用完整 URL
- 链接文本应描述性强

## 质量检查

在提交文档前，运行以下检查：

```bash
# 检查 Markdown 语法（需要安装 markdownlint）
markdownlint docs/zh/**/*.md

# 检查链接有效性（需要安装 linkchecker）
linkchecker docs/zh/
```
