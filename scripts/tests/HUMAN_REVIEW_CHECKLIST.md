## 附录：审查指南

### 如何使用此检查清单

1. **打印或复制此清单**，为每次审查创建新副本
2. **按顺序进行审查**，不要跳过任何部分
3. **记录所有问题**，即使是小问题
4. **提供具体建议**，而不仅仅是指出问题
5. **评估优先级**，帮助团队决定修复顺序
6. **完成总结**，提供整体评价和建议

### 审查最佳实践

1. **从用户角度审查**: 假设自己是第一次使用文档的用户
2. **实际测试示例**: 不要只是阅读代码，要实际运行
3. **检查一致性**: 确保术语、格式、风格在所有文档中一致
4. **关注细节**: 小错误也会影响用户体验
5. **提供建设性反馈**: 不仅指出问题，还要提供解决方案

### 常见问题检查点

- [ ] 代码示例是否包含必要的导入语句？
- [ ] 配置示例是否包含所有必需的字段？
- [ ] 命令示例是否包含必要的权限说明？
- [ ] 错误消息是否有相应的解决方案？
- [ ] 外部链接是否仍然有效？
- [ ] 版本信息是否最新？
- [ ] 截图（如有）是否清晰且最新？
